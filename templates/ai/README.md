# AI-facing scaffold (ai-rules)

Goal: make the codebase legible to AI collaborators while keeping token usage and write scope predictable. This folder is optional but recommended for any BEAM project that wants Codeflow-style selective context.

## Layout
- `context/` — machine-readable snapshots an agent can fetch without scanning the whole tree (routes, Ash resources, schemas). Generated by `mix ai.dump`.
- `policies/` — guardrails for tools. `allowlist.txt` lists glob patterns that agents may write to; adjust per project.
- `prompts/` — reusable prompt snippets for plan/build/review tasks.
- `notes/` — short rotating breadcrumbs (decisions, TODOs) that agents can read before acting.

## Workflow
1) Run `mix ai.dump` after schema/route/Ash changes to refresh JSON manifests under `ai/context/`.
   - Scripts equivalent: `mix run scripts/ai/dump_context.exs` (legacy).
2) Agents should prefer manifest + span fetches (small slices) instead of whole-file reads.
3) Writes go through `scripts/ai/apply_patch.exs` (allowlist enforced) or `/fs/patch` when the dev Plug is enabled.
4) Tests go through `mix ai.test` (or `scripts/ai/run_tests.exs`) to keep scope tight.
5) Optional API server for agents: `AI_AGENT=1 MIX_ENV=dev mix run scripts/ai/serve_agent.exs` (Bandit HTTP on `AI_AGENT_PORT` or 4040) serving the endpoints in `TestProject.AiAgentAPI`. Set `AI_AGENT_SECRET` to require `x-ai-agent-secret` header.
6) Optional stdio server: `AI_AGENT=1 MIX_ENV=dev mix run scripts/ai/serve_agent_stdio.exs` for tools that prefer stdio over HTTP. If `AI_AGENT_SECRET` is set, include it as `secret` in each request object.

## Optional dev-only Plug (clarification of option 5)
If you want HTTP access instead of CLI scripts, add a tiny Plug pipeline (e.g., `AiRules.AgentAPI`) mounted only in `dev` with an env flag (`AI_AGENT=1`). Endpoints: `/ctx/routes`, `/ctx/ash`, `/fs/read?path&start&end`, `/fs/patch`, `/test`. It should:
- Require an allowlist (same `ai/policies/allowlist.txt`).
- Deny writes by default; allow per path.
- Return small JSON payloads for routes/resources/tests.
- Be disabled in prod.

That Plug is not added yet; this doc just clarifies the intent.
